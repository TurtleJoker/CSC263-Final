Database Design Schema

IncidentTypes(typeID, typeName)
typeID is primary key

Incidents(incidentID, typeID, date, state(opened/closed/stalled), comments)
incidentID is primary key

Jobs(jobID, jobTitle)
jobID is primary key

Teams(teamID, teamName)
teamID is primary key

People(personID, firstName, lastName, jobID, teamID, email)
personID is primary key

Incidents_IPAddresses(ipAddress, incidentID, handlerID, reason)
ipAddress is the primary key.

Handler(handlerID, incidentID, personID, reason)
handlerId is primary key

Incidents_Comments(commentID, IncidentID, handlerID, comments, date)
commentID is primary key
